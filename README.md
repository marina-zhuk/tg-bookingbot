# FitAdmin Bot

**FitAdmin Bot** — портфолио-пример Telegram-системы для фитнес-клуба, студии или тренера.

Проект показывает, как может выглядеть система, где клиент открывает Telegram-бота, выбирает абонемент или услугу, проходит demo-оплату, а администратор управляет клиентами, абонементами, рассылками и отчётами.

## Важно

Это **не готовая коробка “купил и забрал”**.

Это витринный demo-проект: пример того, какую систему можно сделать **под конкретный бизнес** — с его услугами, ценами, брендом, оплатами, расписанием, базой клиентов и внутренними процессами.

## Что демонстрирует проект

FitAdmin Bot показывает Telegram-автоматизацию для бизнеса, где есть услуги, абонементы, повторные клиенты и администратор.

В demo-сценарии есть:

- регистрация клиента в Telegram;
- каталог абонементов и спецпредложений;
- demo payment flow без реального списания денег;
- admin panel внутри Telegram;
- управление абонементами, ценами и видимостью;
- список клиентов и оплат;
- Excel/XLSX-отчёт;
- рассылки по пользователям бота.

## Для кого можно адаптировать

Такую систему можно собрать под:

- фитнес-клуб;
- йога/пилатес студию;
- танцевальную школу;
- тренера;
- спортивную секцию;
- beauty/массажный кабинет;
- локальный сервис с пакетами услуг и повторными клиентами.

## Как это объяснять клиенту

Короткая формулировка:

> Это пример Telegram-системы для фитнес-бизнеса. Я могу сделать похожую под ваш бизнес: с вашими услугами, ценами, заявками, оплатами, админкой, рассылками и отчётами.

Ещё проще:

> Клиент всё делает в Telegram, а владелец видит заявки, оплаты и клиентов в админке.

## Что получает бизнес

- Клиенты не теряются в переписках.
- Услуги и абонементы собраны в одном месте.
- Часть вопросов и действий переносится в Telegram-бота.
- Администратор быстрее видит клиентов и оплаты.
- Можно делать рассылки по базе пользователей.
- Можно выгружать отчёты.
- Систему можно адаптировать под реальные процессы бизнеса.

## Demo vs production

### Сейчас в demo

- Показан рабочий Telegram flow.
- Используются demo-данные.
- Оплата работает как mock/demo flow — без списания реальных денег.
- SQLite используется как MVP/demo-хранилище.
- Vercel используется как витрина/landing и demo endpoint.

### Для реального клиента отдельно подключается

- Реальная оплата: YouKassa, CloudPayments или другой provider.
- Production database: PostgreSQL, Supabase, Neon или VPS-хранилище.
- Надёжный webhook hosting.
- Бэкапы, мониторинг и логи.
- Юридические тексты по оплате, возвратам и персональным данным.
- Брендинг, услуги, цены, сценарии и тексты конкретного клиента.

## Для разработчика

Технически проект построен на:

- Node.js CommonJS;
- Telegraf;
- Telegram Bot API;
- Express;
- SQLite через `node:sqlite`;
- `exceljs`;
- Vercel static landing + serverless API endpoints.

Проект требует Node.js `>=22.0.0`, потому что используется `node:sqlite`.

Для portfolio demo это нормально. Для production-клиента лучше использовать persistent database или VPS-инфраструктуру с бэкапами и мониторингом.

## Безопасность

Реальные Telegram tokens, admin IDs, payment keys, webhook URLs, базы, логи, телефоны и email клиентов нельзя коммитить в репозиторий.
