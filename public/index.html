Features

  • Каталог абонементов и спецпредложений
  • Demo payment flow без списания денег
  • Admin panel, CSV export и broadcast
  • Управление абонементами и ценами
  • Подготовка к production YooKassa setup

Stack

  • Node.js
  • Telegraf
  • Express
  • SQLite
  • YooKassa SDK
  • PM2 config

User Flow

/start открывает регистрацию
Пользователь вводит ФИО, телефон и email
Выбирает абонемент из каталога
Проходит demo payment flow
Получает confirmation message

Demo Mode

Portfolio version runs with TEST_MODE=true. Real YooKassa payments are not connected in this demo, and real money is not charged.

For production, the project needs webhook hosting, YooKassa credentials, legal checks, monitoring and backups.

On Vercel, the bot uses /api/telegram as a webhook endpoint for demo mode. Runtime data is temporary and should be moved to an external database for production.

This Vercel deployment includes a portfolio preview page and a demo Telegram webhook endpoint. For production, move persistence to an external database and complete the webhook/payment setup.
